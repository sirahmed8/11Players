# 11Players — Comprehensive Overhaul Plan

## Summary of All Findings from Deep Analysis

### Bugs Found
1. **PlayerRatingModal writes fail silently** — `firestore.rules` has NO rule for `communities/{cid}/matches/{mid}/ratings/{rid}` subcollection → default-deny. Ratings never save.
2. **SiteRatingModal writes also fail** — No rule for `site_feedback` collection. Rating appears sent but data never reaches Firestore.
3. **Profile redirect bug** — Clicking a user's "Profile" from /users navigates to `/profile?uid=X` correctly, but the empty `src/app/profile/[uid]/` folder + static export + `effectiveUid = uid || user?.uid` fallback can lose the query param, redirecting to self.
4. **`?tab=history` not parsed** — Sidebar rate-match toast links to `/match?tab=history` but `match/page.tsx` never reads `searchParams`, always opens on "current" tab.
5. **Notification timestamps inconsistent** — 5 call sites use `new Date()` or `.toISOString()` (string), 4 use `serverTimestamp()` (Firestore Timestamp). Brittle parsing.
6. **Leaderboard "loads 2 then rest" on mobile** — `stats/page.tsx` slices `topThree` and collapses `remainingPlayers` behind an expand button. Default collapsed = only 3 rows visible.
7. **`?link=/profile` in notifications doesn't include uid** — Multiple notifications link to `/profile` (without `?uid=`), always opening own profile instead of the relevant player's.
